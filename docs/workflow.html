Detailed Steps

Install track once

Get the binary onto your PATH. No dependencies, no runtime needed.

# Windows (copy to a folder in PATH)
copy track\bin\track.exe %USERPROFILE%\bin\track.exe

# Or Mac
cp track/bin/track-darwin-arm64 /usr/local/bin/track

Start the server once

Launches in background, auto-creates database at ~/.track/track.db

track serve
# → Track: http://localhost:3012 (PID 12345)

Guided kickoff once

The onboard-project skill guides you through a structured project setup. It gathers requirements, defines your release milestones (epics), and identifies features under each.

# Skill walks through the full kickoff:
/onboard-project

What the kickoff covers:

# 1. Create the project
track project create PROJ "My Project"

# 2. Gather requirements (through conversation)
# - What problem are you solving?
# - Who are the stakeholders?
# - What data sources exist?
# - What are the constraints/deadlines?

# 3. Define epics (release milestones)
track task create --project PROJ --title "POC - Core pipeline" --type epic --priority urgent
track task create --project PROJ --title "MVP1 - Dashboard + alerting" --type epic --priority high
track task create --project PROJ --title "MVP2 - Self-service" --type epic --priority medium

# 4. Identify features under each epic
track task create --project PROJ --title "Data ingestion pipeline" --type feature --parent PROJ-1
track task create --project PROJ --title "Quality scoring model" --type feature --parent PROJ-1
track task create --project PROJ --title "Real-time dashboard" --type feature --parent PROJ-2
track task create --project PROJ --title "Alert rules engine" --type feature --parent PROJ-2
At this point you have: Project created, requirements documented, epics defined as release milestones, features identified under each epic. No implementation tasks yet — those come from /decompose when you're ready to build a feature.

Set up ADO sync (optional) optional

Pull work items from Azure DevOps. Requires a PAT with "Work Items: Read" scope.

# Set PAT in environment
$env:TRACK_ADO_PAT = "your-pat-here"

# Create config template
track ado config

# Edit ~/.track/ado.json with your org/team/email
# Then pull:
track ado pull

Decompose first feature once

When ready to start building, decompose your first feature into implementation tasks.

# Decompose the highest-priority feature:
/decompose PROJ-4

# Creates sized, sequenced tasks like:
# PROJ-8: Set up ingestion config (S, 0.5h)
# PROJ-9: Implement source connector (M, 1.5h)
# PROJ-10: Add validation rules (M, 1.5h)
# PROJ-11: Write integration tests (S, 1h)

Setup complete — enter daily loop

From here, workflow matches Scenario 3 (Active Project)

Install track once

Same as Scenario 1 — get the binary, start the server.

track serve

Run onboard-project once

Imports existing backlog and organizes it into the epic/feature/task hierarchy.

# Skill does all of this automatically:
/onboard-project

What it does:

# 1. Creates project
track project create PROJ "My Project"

# 2. Reads existing sources:
# - docs/sprint-tasks.md
# - CLAUDE.md backlog section
# - docs/session-notes/ (recent context)

# 3. Maps existing items to hierarchy:
# - Groups related items → creates epics
# - Identifies capabilities → creates features
# - Preserves existing tasks + statuses
track task create --project PROJ --title "Phase 1 - Current sprint" --type epic
track task create --project PROJ --title "Dashboard improvements" --type feature --parent PROJ-1
track task create --project PROJ --title "Fix chart rendering" --type task --parent PROJ-2
track task move PROJ-3 --status in_progress

# 4. Adds ## Taskboard section to CLAUDE.md

Estimate unsized tasks once

Imported tasks likely lack size estimates. Bulk-estimate the backlog.

/estimate --project PROJ

Set up ADO sync (if applicable) optional

Same as Scenario 1. Links existing ADO work items to track tasks.

track ado config
# edit ~/.track/ado.json
track ado pull

Conversion complete — enter daily loop

From here, workflow matches Scenario 3 (Active Project)

Session start every session

Orient: see board state, in-progress tasks, blockers, suggested next steps.

# Skill reads CLAUDE.md + queries track automatically
/session-start

Or manually:

track task list --project PROJ
track session start --project PROJ --branch $(git branch --show-current)
Session start shows: in-progress tasks, ready tasks by priority, blockers, active sprint progress, uncommitted git changes.

Pick up a task every session

Move the next task to in_progress. If the next feature hasn't been decomposed yet, run /decompose first.

# Auto-pick highest priority ready task:
track task next

# Or manually:
track task move PROJ-5 --status in_progress

# If no ready tasks — decompose next feature:
/decompose PROJ-12

Do the work every session

Write code, run tests, iterate. Track stays out of your way.

# If new work discovered mid-session:
track task create --project PROJ --title "Found edge case in X" --priority high --source discovered

# If blocked:
track blocker create --title "Waiting on API access" --project PROJ

Complete tasks every session

Mark done, log actual time.

track task done PROJ-5 --actual-hours 1.5

# Or log time separately:
track session log PROJ-5 --hours 1.5 --note "Implemented retry logic"

End session every session

Wraps up: updates task states, logs time, writes session notes, updates CLAUDE.md.

# Skill does all of this:
/session-end

Or manually:

track session end --summary "Completed retry logic for pipeline ingestion"

Plan sprint weekly

At start of week: select work from decomposed features, sequence tasks, set capacity targets.

/plan-sprint --project PROJ

Creates a sprint in track with selected tasks and sequencing.

Weekly report weekly

End of week: summarize progress, velocity, blockers, next week plan.

/weekly-report

Project status as needed

Cross-project dashboard for standups or orientation.

/project-status

Decompose next feature as needed

When current tasks are done and you're ready to start the next feature, decompose it into tasks.

# Decompose a feature into implementation tasks:
/decompose PROJ-20

# Estimate if needed:
/estimate --project PROJ
Decompose on demand: Don't decompose all features upfront. Decompose the next feature when you're ready to work on it — requirements may have changed by then.

ADO sync as needed

Pull latest work items from Azure DevOps when assignments change.

track ado pull
track ado status

Timeline projection as needed

Forecast completion dates for stakeholder communication.

/timeline --project PROJ
